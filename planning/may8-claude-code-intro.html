Claude Code

AI that works inside your project


APHYS AI Tutorial · May 8, 2026 · Module 2

This is not a chatbot

Claude.ai answers questions. Claude Code does work — it reads your files, runs code, checks results, and iterates until the task is done.

Claude.ai (chat)

  • You paste text
  • It replies with text
  • You copy-paste the result
  • Starts fresh every time

Claude Code (agent)

  • Reads your actual files
  • Runs Python, shell, git
  • Writes results directly to disk
  • Remembers your project context

Think of it as a capable lab assistant who has read every file in your project and can type in your terminal.

The Agentic Loop

Claude works in a continuous loop — not a single reply. You can interrupt and steer at any point.

🔍 Gather Read files,
search code
→
⚙️ Act Edit, run,
write output
→
✅ Verify Check result,
run tests
→
💬 Report Done or
ask you
↺   repeats until task is complete

Press Esc to interrupt anytime. Press Esc Esc to rewind file changes.

What Claude Code can see

Start it in your project folder — it reads the whole tree and knows your context.

Your files

Python scripts, Jupyter notebooks, CSV data, figures, LaTeX drafts — everything in the folder

Your terminal

Runs python, conda, git, pip — any command you can type

The web

Can search for documentation, fetch papers, look up API references on demand

CLAUDE.md

A file you write — project context, conventions, what not to do. Loaded every session.

Auto memory

Notes Claude writes to itself as you work — your preferences, discovered patterns.

Getting Started

1. Install (one command)

# macOS / Linux
curl -fsSL https://claude.ai/install.sh | bash

# Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex

2. Start in your project folder

cd ~/my-experiment
claude

3. Log in with your Claude account (first time only)

# If you have a KTH Claude Team seat — just log in and it works
# If not, request one at: aphys-ai.kth.se → "Claude Team Seat"

Also available as a Desktop App (no terminal needed) and inside VS Code or JetBrains IDEs.

You are always in control

Claude asks before making changes. You decide how much autonomy it has.

Default

Asks permission before every file edit and command. Good for learning.

Auto-accept

Edits files freely, still asks for shell commands. Good when you trust the task.

Plan mode

Read-only — analyses everything, changes nothing. Great for exploration.

Rewind any change

Every file edit is snapshotted. Press Esc Esc to roll back. Or just ask: "undo the last change".

Switch modes

Press Shift+Tab to cycle between Default → Auto-accept → Plan mode at any time.

Workflow: Analyse your data

A real example — three prompts to go from raw spectra to a report.

1. Navigate and start

cd ~/experiments/raman-2026-05
claude

2. Describe the task in plain language

I have 48 Raman spectra in spectra/*.csv (columns: wavenumber, intensity).
Load them all, subtract a polynomial baseline, find peaks above 3-sigma noise,
and fit each peak with a Lorentzian. Save a summary table as peaks.csv
and a figure per sample in figures/.

3. Steer as it works

The fits in the 1200–1400 cm⁻¹ region look off — try a tighter fit window

You don't need to write perfect prompts. Describe your goal, watch it work, correct course naturally.

Your turn — 15 min hands-on

Open Claude Code in your terminal (or claude.ai in your browser). Pick the track that fits where you are.

Track A — Browser only  (no install needed)

Open claude.ai, start a new conversation, and type:

I have a folder of CSV files from my lab. Each file has columns:
time (s), signal (a.u.). Describe step by step how you would help me
load them, subtract background, and plot all curves overlaid.
Don't write code yet — just explain your approach.

Then ask it to write the code, and keep iterating.

Track B — Claude Code in terminal  (recommended)

Navigate to any folder with data or code on your laptop, then type:

cd ~/your-project-folder
claude
> Give me an overview of what's in this folder.
  Then suggest 3 things you could help me do with these files.

Follow one suggestion and see how far it gets in 10 minutes.

Track C — Advanced  (if you're already comfortable)

> I want to automate [describe a repetitive task in your lab workflow].
  Help me design a script that handles this end-to-end.
  Start by asking me questions to understand the task fully.
