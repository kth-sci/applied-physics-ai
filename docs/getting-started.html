A practical introduction for Applied Physics faculty -- what to use, why, and how to begin.

AI for Scientists: The Practical Landscape

You do not need to understand transformer architectures to use AI effectively. Here is what matters.

As physicists, you already work with computational tools daily -- from numerical simulations to data analysis scripts. Modern AI tools are best understood as a new category of computational instrument: systems that can read, write, reason about, and generate text, code, and structured analysis. The key shift is that you interact with them using natural language rather than formal syntax.

There are hundreds of AI tools available. We recommend starting with exactly two, chosen because they complement each other and are already accessible at KTH:

Start Here

ChatGPT

What it is: A conversational AI by OpenAI. You type questions or instructions in natural language; it responds with text, code, analysis, or creative content.

Best for:

  • Quick questions and explanations
  • Brainstorming and ideation
  • Drafting text (grants, emails, reports)
  • Summarizing papers and documents
  • Simple code generation and debugging
  • Data interpretation and analysis guidance

Access: KTH has procured institutional ChatGPT licenses. Log in with your KTH account at chatgpt.com.

Recommended plan: ChatGPT Plus (included in KTH procurement) gives access to GPT-5.4, the most capable model, plus file uploads, image analysis, and advanced data analysis.
Power Tool

Claude Code (AI Agent)

What it is: An AI agent by Anthropic that runs in your terminal. Unlike a chatbot, it can autonomously read files, write code, run commands, search the web, and manage multi-step tasks.

Best for:

  • Writing and debugging complex code
  • Analyzing large datasets end-to-end
  • Building web applications and dashboards
  • Automating multi-step research workflows
  • Working with repositories and codebases
  • Converting MATLAB/Fortran to modern Python

Access: Requires a personal Claude subscription from Anthropic.

Recommended plan: Claude Pro ($20/month) or Claude Max ($100/month for heavy use). The Pro plan is sufficient for most faculty needs.

Key Concepts for Scientists

A mental model for understanding modern AI tools.

An LLM is a neural network trained on vast amounts of text data. It learns statistical patterns in language -- essentially, given a sequence of words, it predicts what comes next. But these "predictions" are so sophisticated that the model can write code, explain physics, translate languages, and reason about problems.

Analogy: Think of it like a very large basis set expansion. The model has learned to represent and combine concepts in a high-dimensional space, much like how you can represent any function as a superposition of basis functions. The more parameters (basis functions), the richer the representation.

Chatbot (e.g., ChatGPT): You send a message, it replies. It is reactive -- it responds to each prompt independently (or with conversation context). It cannot take actions on its own.

AI Agent (e.g., Claude Code): You describe a goal, it plans and executes a sequence of actions autonomously. It can read files, write code, run commands, search the web, and iterate on its own results. It is like the difference between asking a colleague a question vs. delegating a task to a research assistant.

Prompt engineering is the art of giving AI clear, effective instructions. Just as the quality of a scientific question determines the quality of the experiment, the quality of your prompt determines the quality of the AI's output.

Key principles: be specific, provide context, specify the output format you want, and iterate. See our ChatGPT Quick-Start Guide for a complete cheat sheet.

Hallucination: AI can generate plausible-sounding but incorrect information -- including fake references, wrong equations, and incorrect numerical results. Always verify quantitative claims.

No true understanding: The model does not understand physics the way you do. It can pattern-match and reason statistically, but it lacks the physical intuition built from years of lab work and theoretical study.

Context limits: Models have a finite "context window" -- the amount of text they can consider at once. Very large documents or codebases may need to be processed in parts.

Your domain expertise is the quality filter. AI is fast but not always right. The combination of AI speed and your scientific judgment is what makes this powerful.
