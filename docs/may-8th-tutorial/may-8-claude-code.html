Foundation

What is an AI agent?

A language model that doesn't just answer — it decides what to do and does it.

Perceive — reads the world

Files, code, data, PDFs, terminal output, web pages — the agent observes its environment before acting.

Plan — breaks goals into steps

Given a high-level goal, it decides which tools to use, in what order, and checks the result before continuing.

Act — uses tools

Writes files, runs code, calls APIs, controls instruments. Output lives on disk — not in a chat window.

Remember — persists knowledge

Context window for this session, CLAUDE.md for every session. Memory you design and control.

Iterate — loops until done

Observe → Plan → Act → Observe → … Reads errors, fixes them, re-runs. You intervene when it asks.

Harness engineering: structuring the context, memory, and tools that shape agent behavior. Beyond prompting — you design the environment the agent operates in.
Chatbot: text in, text out.
Agent: goal in, result on disk.

Recap

You already know the key idea.

ChatGPT

Text in. Text out.

  • Replies once, then waits
  • No access to your files or disk
  • No memory between sessions
  • You copy-paste everything manually
  • Output lives only in the chat window
Claude Code (AI Agent)

Goal in. Result on disk.

  • Reads your files — code, data, PDFs, notebooks
  • Runs terminal commands — python, bash, git, conda
  • Saves output directly to disk — figures, reports, scripts
  • Loops until the task is done — plan → act → fix → repeat
  • Persists instructions across sessions via CLAUDE.md
The fundamental shift: Claude Code doesn't produce text for you to copy. It produces files on your disk, code that runs in your environment, and results you can open immediately.

Foundation

Start with a folder, not a chat window.

Claude Code's world is a directory. Everything it knows, reads, writes, and remembers lives there.

Files it reads
Code, data, docs, PDFs
Python scripts, CSV files, Jupyter notebooks, papers, notes — anything in the directory tree
Commands it runs
python, bash, git, pip
Any command your shell can run — conda environments, custom scripts, SLURM job submissions
Output it saves
Figures, reports, scripts
PNG figures, markdown reports, HTML pages, edited source files — real artifacts on your disk
Getting started is one line:
Open a terminal → navigate to your project folder → type claude → Claude reads the directory and knows your project immediately.

Standing Instructions

One file. Every session.

CLAUDE.md is a plain-text file at the root of your project. Claude reads it automatically at the start of every session — without you saying anything.

① Project context

What this folder is for, what data format you use, what tools and libraries are installed.

② Conventions

"Always save figures to ./figures/" · "Use seaborn-whitegrid style" · "Never modify raw data files"

③ Constraints & safety rails

What NOT to do, which directories are off-limits, when to ask before deleting — your guardrails.

④ Generate it instantly

Type /init in any project — Claude writes a CLAUDE.md from your existing files. Edit to add your domain context.

# Pump-Probe Spectroscopy Analysis KTH LAMP group — ultrafast transient absorption data. Data: CSV files in ./data/ columns: time_ps, wavelength_nm, delta_OD Always use matplotlib with seaborn-whitegrid style. Save all figures to ./figures/ as 300 dpi PNG. Never modify files in ./raw_data/. Python environment: conda env 'spectro' (python 3.11, numpy, scipy, matplotlib).
Keep it short. CLAUDE.md loads every session and burns tokens every time. Aim for 10–20 lines. If it's too long, Claude ignores half of it. Use Skills (slide 9) for domain knowledge that only applies sometimes.

The Key Mindset Shift

The question is not: what prompt should I write?
The question is: what should Claude know?

Context engineering — deliberately shaping what's in the context window — is the core skill of working with AI agents.

The Constraint

Context fills fast

Every file read, every command output, every message enters the window. Performance degrades as context fills — Claude starts forgetting earlier instructions.

10 files + 20 messages = context half gone before the real task starts.
Three Levers

Shape what Claude knows

CLAUDE.md — always loaded; @file — read when you mention it; Skills — description pre-loaded (~100 tokens), full body only on trigger.

CLAUDE.md for universal rules · Skills for domain workflows · @file for one-off context.
Rich Context → Rich Output

The compounding effect

Vague prompt + blank context = generic output. Precise task + tight CLAUDE.md + right files = working domain-specific results.

"Analyse my data" → generic. "Read pump_probe.csv, fit 550 nm biexponential, plot" → working figure.
Context Management

Keep it clean

/clear — reset between unrelated tasks. /btw — side question that never enters history. Subagents — investigate in a separate window, report summaries back.

Corrected Claude twice on the same issue? /clear and restart with a better prompt.

The Commands You Need

Daily usage in ten commands.

Start & Run
claude
Start interactive session — reads directory + CLAUDE.md immediately
claude -p "task"
One-shot non-interactive run — pipe into scripts or batch automation
--permission-mode auto
Auto-approve routine actions silently; only prompts for risky steps
--dangerously-skip-permissions
Advanced: fully autonomous — skips all prompts. Use only in git-tracked projects where you can revert. Ideal for overnight or unattended runs.
Build Context
/init
Generate CLAUDE.md from current project — run once, then edit
#add a convention
Append a line to CLAUDE.md without executing — build it incrementally
Manage Context
/clear
Reset context — use between unrelated tasks
/btw [question]
Side question that never enters conversation history
Control & Undo
Esc · Esc+Esc
Interrupt running action · restore previous file state

Extending Claude Code · Part 1

MCP: connect Claude to any external app.

What MCP Is
  • Model Context Protocol — a standard for connecting Claude to external apps
  • Databases, browsers, CAD tools, lab instruments, live APIs
  • Blender, Playwright, SQL, Figma — extensible by design
  • Jonas used it for Blender — controlled 3D objects with natural language
The Cost
  • Connecting an MCP server loads all its tool definitions into context — even unused tools
  • Requires a running server process + config file
  • Complex setup: install → start server → add to config → test connection
  • Token-heavy from the moment it connects
When to Use It
  • Real-time app control — Blender, Figma, live databases where you need two-way state sync
  • Browser automation — Playwright MCP is the best option here
  • No CLI exists and the REST API is complex
  • Rule: if a CLI exists, use the CLI. If you need live bidirectional control, use MCP.
For most research workflows: MCP is overkill. The better default is the combination on the next two slides — Skills + CLI tools. Same power, a fraction of the token cost. registry: github.com/modelcontextprotocol/servers

Extending Claude Code · Part 2

Skills: domain knowledge that costs ~100 tokens.

How Skills work — progressive disclosure

At startup: only name + description loaded (~100 tokens total for all installed skills). When triggered: full SKILL.md body loaded. Reference files: loaded only when needed. Zero cost until relevant.

CLAUDE.md vs Skills — when to use which

CLAUDE.md — loaded every session, always burns tokens. Use only for rules that apply to every single task. Keep ruthlessly short.
Skills — triggered on demand. Use for domain workflows, dataset schemas, instrument conventions, analysis pipelines.

Install or write your own

svamp skills install <name> · agentskills.io · or write a SKILL.md in .claude/skills/<name>/. This tutorial runs on Svamp + Hypha skills.

Minimal SKILL.md for your lab
--- name: spectroscopy-analysis description: Analyze pump-probe TA data: plot transient absorption maps, fit decay curves, export publication figures. Use when working with time-resolved spectroscopy or delta_OD data. --- ## Data format CSV — columns: time_ps, wavelength_nm, delta_OD Raw data in ./raw_data/ — never modify. Figures → ./figures/ at 300 dpi PNG. ## Workflow 1. Read CSV, describe structure and range 2. Plot TA map (wavelength vs time, diverging cmap) 3. Fit decay at requested λ with biexponential 4. Report τ₁, τ₂, amplitudes, R² 5. Save figure as fig_<wavelength>nm_decay.png

The description must say what it does and when to use it — Claude picks from all installed skills using just this field.

The Recommended Stack

For most research tasks:
Skills + CLAUDE.md + CLI tools.

① CLAUDE.md

The permanent brief

10–20 lines. Who you are, what project this is, non-negotiable conventions. Loaded every session. Every token here is always spent — make it count.

eg: "KTH LAMP ultrafast spectroscopy. Python 3.11. Save figures to ./figures/"
② Agent Skills

Domain workflows on demand

Write one Skill per workflow type: spectroscopy analysis, image segmentation, literature synthesis, instrument control. Loaded only when triggered — zero cost otherwise.

eg: "spectroscopy-analysis" triggers on time-resolved data mentions
③ CLI Tools

Native integrations, zero overhead

Claude uses gh, aws, hypha, svamp natively — already authenticated, no server process needed. New tool? Tell Claude: "Use foo --help to learn it, then do X."

Skills + CLI beats MCP for almost every research workflow
Together: a specialized, token-efficient, extensible agent without MCP overhead. CLAUDE.md for identity, Skills for expertise, CLI for actions. Add MCP only when you need live bidirectional app control.

Hands-On · Getting Started

Start your first Claude Code project.

Five steps. By step 4 you'll have a working agent session with a real task running.
Step 1

Create a folder

mkdir my-project
cd my-project

Put any data files you want to use inside it. No data? Claude generates synthetic examples.

Step 2

Optional: git init

git init

Highly recommended — lets you undo Claude's changes with git restore if something goes wrong. Or ask Claude to do it for you.

Step 3

Start Claude Code

claude

Claude reads the folder immediately. Then type /init to generate a CLAUDE.md from your project structure.

Step 4

Describe your project in CLAUDE.md

Edit the generated file. Add 3–5 lines: who you are, what this project does, any conventions. Example:

"I'm a researcher at KTH LAMP studying ultrafast spectroscopy. Goal: analyze pump-probe data in CSV format."

Step 5

Type what you want

Tell Claude what to build in plain language. Pick from slide 13 or bring your own task.

"Read the CSV files in ./data/ and plot the spectra. Save figures to ./figures/."

If you get stuck: paste the error message directly into Claude Code. Don't copy-paste it yourself — just say "read the error and fix it." That's the agent loop.

Hands-On · Choose Your Task

Pick one. Build it. Show it at 14:30.

No data? Tell Claude to generate a synthetic example. These tasks are designed to complete in 10–15 minutes.

LAMP · BON — Spectroscopy / Imaging

Data analysis + figures

Give Claude a CSV or image stack. Ask it to analyze, fit, segment, and export plots. Works for pump-probe spectra, X-ray images, microscopy stacks, or any time-series.

All divisions — Literature

Literature survey

Drop 3–5 PDFs into a folder. Ask Claude to extract key findings, build a comparison table, and flag contradictions. Output: structured markdown you can paste into your paper.

All divisions — Web

Build a webpage

Describe your research group or project in one sentence. Claude builds a single-page HTML site — lab overview, team, publications placeholder. Opens in your browser immediately.

All divisions — Presentation

Create a slide deck

Describe what you want to present. Claude builds a scroll-snap HTML presentation — dark theme, clean typography — you can open fullscreen and show today at the wrap-up.

Professors — Teaching

Design teaching material

Name your course and topic. Claude drafts AI-resistant assignments (lab-based, oral defence, experimental reasoning), lecture exercises, or a problem set with solutions.

All divisions — Writing

Improve a text

Paste an abstract, grant paragraph, or methods section into a .txt file. Ask Claude to rewrite it for a specific audience, flag weak arguments, or suggest citations to add.
