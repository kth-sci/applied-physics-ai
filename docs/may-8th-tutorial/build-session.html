Step 1

Form a group, then brainstorm with ChatGPT

First: find a partner. Groups of 2–3 work best — one person drives Claude Code, the other reviews and directs. Zoom participants: you've been moved into a breakout room with 1–2 others. Introduce yourselves, pick one laptop to work on together.

Once you have a group of 2–3: brainstorm your idea using ChatGPT. Copy the prompt below, paste it in, then have a short conversation to land on something concrete to build. Aim for a project you can get working in ~10 minutes — then use the remaining time to iterate and improve.

Not sure what to bring? Look at the idea list below the prompt — pick anything that matches your work.

ChatGPT brainstorm prompt 
I am a researcher / faculty member at the Department of Applied Physics, KTH Royal Institute of Technology. I am attending a hands-on AI tutorial today and have 45 minutes to build something useful with Claude Code — an AI agent that can write and run code, read my files, and build tools autonomously.

My research area is: [FILL IN — e.g. "ultrafast laser spectroscopy" / "X-ray imaging" / "cell biophysics" / "nanophotonics"]

Help me brainstorm a mini-project that:
- Is achievable in ~10 minutes with an AI agent (you can always go deeper after)
- Is genuinely useful to my actual research, teaching, or daily work
- Uses data, scripts, or documents I can provide (or simple synthetic examples)
- Produces a concrete output: a script, a figure, a document, or a small tool

Ask me up to 4 questions to narrow down the best idea. Then suggest 2–3 concrete options ranked by feasibility in 45 minutes. Once I pick one, write a clear 3-sentence project description I can paste into a CLAUDE.md file.
How to use this: Paste the prompt into ChatGPT. Replace the bracketed field with your research area. Answer the questions it asks. When you have a project description, copy it — you'll use it in Step 3.

Ideas to get you started

Don't have your own data handy? Pick any of these — Claude Code can work with synthetic examples or small files you can download in seconds.

LAMP · BON — spectroscopy

Peak fitting & batch processing

Paste or upload a CSV of spectral data. Claude Code fits Gaussian peaks, identifies anomalies, and writes a batch script for your full dataset. Output: cleaned data + figures.

BON · Biophysics — microscopy

Image analysis script

Provide a few microscopy images (or use sample TIF files). Claude Code writes a Python script to count cells, measure areas, or segment structures. Output: CSV of measurements + annotated images.

All divisions — writing

Research summary generator

Drop in 3–5 of your papers or abstracts. Claude Code extracts key findings, builds a structured summary, and drafts a 1-page narrative for a grant or introduction. Output: a Markdown document.

All divisions — data

Data explorer & visualiser

Give Claude Code any CSV or Excel file from your research. It profiles the data, produces exploratory plots, and suggests what statistical analyses to run next. Output: figures + brief report.

LAMP · BON — automation

Instrument control script

Describe your instrument's API or an existing control script. Claude Code writes a Python wrapper, adds a simple loop for automated acquisition, and documents it. Output: working Python module.

Professors — teaching

AI-resistant assignment design

Tell Claude Code which course and topic. It generates 3 assignment options that require physical lab work, oral defence, or genuine experimental reasoning — things AI cannot shortcut.

All divisions — literature

Paper comparison table

Paste 3–5 abstracts or short excerpts from related papers. Claude Code builds a structured comparison table: methods, datasets, key results, limitations. Output: Markdown table ready for your paper.

All divisions — productivity

Repetitive task automator

Describe something you do repeatedly: rename files, reformat data, generate weekly reports. Claude Code writes a script that does it in one command. Output: a shell or Python script you can reuse.

Step 3

Write your CLAUDE.md

The CLAUDE.md file is how you give Claude Code the context it needs to act as a useful collaborator rather than a generic assistant. Think of it as a briefing document for a new team member who knows a lot about AI but nothing about your specific work.

Fastest path: let the agent write it. Open Claude Code, navigate to your project folder, and paste this:
"I'm a [role] at KTH Applied Physics working on [your field]. I want to build [one sentence description]. Write a CLAUDE.md for this project and save it."
Claude will ask any clarifying questions it needs, then write and save the file for you.

Or use the template below as a starting point and edit it yourself:

CLAUDE.md template — copy and edit 
# Project: [Short name — e.g. "ARPES Peak Fitting" or "Cell Counter"]

## Who I am
I am a [researcher / postdoc / professor] at the Department of Applied Physics, KTH.
My research focuses on [your field — e.g. "ultrafast photoemission spectroscopy" / "X-ray phase contrast imaging"].

## Goal
[Paste your 2–3 sentence project description from ChatGPT here, or write it yourself.]

Example: "Build a Python script that reads CSV files of pump-probe spectra, fits
Gaussian peaks, extracts peak positions and widths, and produces a summary figure
comparing spectra across different time delays."

## Data and files
- [Describe what files are in this folder]
- Example: "spectra.csv — 50 columns of intensity vs. energy, 1 column of time delay labels"
- Or: "No data yet — please generate a realistic synthetic example to test with"

## Output I want
[What should the final result look like?]
- Example: "A Python script I can run from the terminal, plus 2–3 figures"
- Example: "A Markdown file with a structured literature comparison table"

## Constraints
- Time: aim for a working result in ~10 minutes; iterate from there
- Keep it simple — prioritize a working solution over a perfect one
- Use Python unless there is a strong reason not to
- Add brief comments to any code so I understand what it does
Tip: You can also ask ChatGPT to write the CLAUDE.md for you. Paste this into ChatGPT after your brainstorm: "Now write a complete CLAUDE.md file for this project based on what we discussed. Use the template format with sections: Who I am, Goal, Data and files, Output I want, Constraints."

Save the file as CLAUDE.md inside your project folder. That's it — Claude Code will read it automatically when it starts.

Step 4

Start building with Claude Code

Open Claude Code and point it at your project folder. Then paste the starting prompt below. This is where the hands-on work begins.

CLI (terminal)
Terminal
cd ~/my-project
claude
Claude Desktop — Code tab
  1. The folder you opened in Step 2 is already loaded
  2. Paste the starting prompt directly into the chat box
  3. Claude Desktop reads your CLAUDE.md automatically

Once Claude Code starts, paste this as your first message:

Starting prompt for Claude Code 
Read the CLAUDE.md file in this folder carefully. This is your briefing for what we're building together today.

Before starting, ask me any clarifying questions you have — especially about the data format, what the output should look like, and any constraints I haven't mentioned.

Once you understand the task, propose a concrete step-by-step plan (maximum 5 steps). Wait for my approval before writing any code.

Remember: aim to have something working in 10 minutes. Work as a group — one person drives Claude Code, the other(s) review and direct. Prioritize a working result over a perfect one.
What to expect: Claude Code will read your CLAUDE.md, ask a few questions, then propose a plan. Review it — you can ask it to adjust. Once you approve, let it work. It will write code, run it, fix errors, and iterate. Your job is to review the output and apply your domain judgment.

Using the techniques from today

Manager mindset — don't micromanage

Let it work, then review

After your starting prompt, step back. Let Claude Code loop through the task — it will think, write code, run it, check the output, and fix errors on its own. Interrupt only if it asks a question or goes in the wrong direction. Review the result with your domain knowledge.

Context is everything

If it's not working, add more context

If Claude Code produces something wrong, don't just say "that's wrong." Explain why it's wrong in terms of your domain: "In ARPES data, the background is never negative — this fit is unphysical. Try a different baseline model." Your physics knowledge is the correction signal.

Use ChatGPT as a debugger

Stuck? Ask ChatGPT first

If Claude Code produces an error you don't understand, paste the error message into ChatGPT and ask "what does this error mean and how should I fix it?" Then bring the answer back to Claude Code. Use both tools together.

Iterate, don't restart

Refine what you have

When the first version is done, ask for improvements: "The figure is too small and the axis labels need units." or "Can you add a command-line argument so I can run this on any file?" Small iterations build on what works.

Step 5

Share what you built

At 16:45 we'll come back together for a short show & tell. You don't need to have finished — even a first draft is worth sharing. A partial result with good insight is more valuable than a polished one with nothing to say about the process.

Generate your presentation slides

Each group gets ~2 minutes. Ask Claude Code to make you a simple 2-slide HTML deck — it takes under a minute and gives you something clean to project. Paste this prompt when your project is done (or mostly done):

Prompt — generate presentation slides 
Create a single-file HTML slide deck I can open in a browser and present in 2 minutes. Make it 2 slides with scroll-snap (each slide fills the full viewport). Dark background, large readable text, Inter font.

Slide 1 — What I built
- Title: [project name]
- What it does: 1–2 sentences
- Key output or result (describe it, or embed a figure if one exists in this folder)

Slide 2 — What I learned
- What surprised me or worked unexpectedly well
- One thing I'd do next with more time

Keep the design clean and minimal — readable from across a room. Save it as presentation.html in this folder.
Then open it: Claude Code saves presentation.html in your project folder. Open it in your browser (double-click the file, or open presentation.html in the terminal), go fullscreen (F11 / Cmd+Shift+F), and you're ready to present. Desktop users: find the file in Finder/Explorer and double-click it.

Show & tell format

Option A — show live

Demo in the room

Open your presentation.html (or show the actual output). Walk through what the agent did, what surprised you, and what you'd do next. 2 minutes per group.

Screen share or chat

Screen-share your slides or output directly. Or paste a screenshot in the Zoom chat if you'd rather not present live.
