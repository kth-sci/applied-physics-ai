The shift

AI agents ≠ chatbots

The jump from language model to agent is a fundamental shift in what AI can do for you.

Before — Chat model

Ask → Answer → Done

Single-turn conversation. No memory. No actions.

  • You type a question, it responds in text
  • No memory between sessions
  • Cannot execute code, read files, or call APIs
  • Great for drafting, explaining, summarising
  • Accuracy depends entirely on training data
Now — AI Agent

Plan → Act → Observe → Repeat

Autonomous multi-step loops with real-world tools

  • Reads your files, git history, and terminal output
  • Executes code, installs packages, fixes its own errors
  • Runs command-line tools and scripts via agent skills
  • Delegates sub-tasks and tracks progress
  • Asks clarifying questions when it needs context
Step 1

Think

Reason about the goal and plan sub-tasks

→
Step 2

Use tool

Run code · read file · call API · search web

→
Step 3

Observe

Read the result and check against the goal

→
Step 4

Repeat

Iterate until done — or ask the human

→
Done

Deliver

Return result to you for review and judgment

Agent skill

Markdown + instructions

A plain text file describing a capability — what the skill does, when to use it, and which commands or scripts to run. The agent reads it and gains the ability.

Example — data analysis

analyse-spectra.md

# Usage: python fit_peaks.py <file>
Fits Gaussian peaks to ARPES data.
Outputs: peaks.csv + figure.png

Example — lab control

control-stage.md

# Usage: python stage.py move x y
Controls motorised stage via USB.
Returns: position confirmation

Economics

Software is becoming zero cost

This is not a metaphor. It is a structural economic change that affects how research is produced and who produces it.

Code

Marginal cost → zero

A working Python analysis pipeline that would take a grad student two weeks now takes 30 minutes with Claude Code. The marginal cost of functional software is approaching zero.

  • Batch processing scripts in minutes
  • Full analysis pipelines from data to figure
  • Instrument control code on demand
Knowledge

Synthesis is cheap

Literature review, summarisation, and first-draft writing are extremely cheap. The expert who only holds information is becoming vulnerable. The expert who judges is not.

  • 100-paper syntheses in minutes
  • Grant narrative first drafts in hours
  • Comparison tables from many sources
Expertise gap

The thinning wall

The gap between "junior" and "senior" output narrows fast when AI amplifies junior researchers' productivity. Domain judgment becomes the scarcest and most defensible resource.

  • Physical intuition cannot be synthesised
  • Experimental design still needs deep knowledge
  • Scientific taste is human
Opportunity: A single researcher with good domain knowledge and AI fluency can produce what previously required a full team. Research output per person can scale dramatically.
Risk: If everyone uses AI without critical review, we get faster production of mediocre work. Quality gates — peer review, physical intuition, reproducibility — matter more, not less.

The fundamental shift

From tool user to manager of AI

The same AI agent. Completely different outcomes. The gap between good and bad use is not the model — it is the person directing it.

Tool-user mindset

I ask, it answers

  • "Write me a Python script for peak fitting" — then nothing works and you give up
  • Paste error → ask what it means → fix one line → hit next error → repeat forever
  • Micromanage every step — approve each file, check each command
  • Frustrated when it doesn't "just do what I mean"
  • Every session starts from scratch with zero context about your work

Result: 1× output. Matches what you could do alone, just slightly faster.

Manager / PI mindset

I set goals, it executes

  • Provide rich context: "We use time-resolved ARPES. Data is in /data/arpes/. Goal: batch-process all runs, flag outliers, produce summary figure."
  • Grant real autonomy: let it loop for 20 minutes without interrupting
  • Think from the agent's perspective: what would you need to know to do this task well?
  • Give it skills and tools: access to scripts, instrument APIs, agent skill files
  • Review critically at the end: your physical intuition is the quality gate

Result: 10–100× output. The multiplier comes from how you communicate, not the model.

Communication
New superpower
How clearly you can describe goals, constraints, and context is now the bottleneck — not coding skill.
Judgment
Multiplies output
Good judgment about when to trust the agent, when to redirect, and when to override — this is the irreplaceable skill.
Autonomy
Let it work
Micromanaging every step costs you the benefit. Set the goal clearly, then step back and let the loop run.
Physics
Your edge
Your domain knowledge is what makes the output correct — not just produced. AI amplifies expertise; it doesn't replace it.

Teaching

The teaching disruption is unresolved

This is not a future problem. It is happening in your courses right now, this semester.

What is broken

The old model no longer holds

  • Take-home assignments — AI completes them in seconds. Submitted work may reflect zero learning.
  • Traditional coding labs — students run AI output without understanding what it does or why.
  • Productive struggle — instant answers remove the friction that builds deep understanding.
  • Written reports — indistinguishable from AI-generated content. Detection tools fail.
  • Junior mentorship — if AI does the learning tasks, where does expertise transfer?
What might work

Design for the AI era

  • Oral examinations — students defend AI-generated work in person. Shallow use becomes immediately visible.
  • Lab-access requirements — assignments that require physical lab work, real data, and in-situ decisions AI cannot make.
  • AI-as-collaborator tasks — students direct an agent, justify choices, and critique outputs. Builds the PI mindset early.
  • Process portfolios — document how you arrived at a result, not just what it is. Shows thinking.
  • Peer teaching — explain your work to a peer, not to a rubric. Forces real understanding.
We don't have all the answers. The June 15 APHYS Teachers' Meeting will focus entirely on this. Today is about building shared vocabulary and early practice so that conversation is grounded in experience.

Responsibility

Responsible use in research

AI is entering every phase of the research cycle. The risks are not hypothetical — they show up in peer review, reproducibility, and scientific quality.

AI is used across the full research cycle

①

Idea generation & hypothesis

AI suggests research directions, experimental designs, and gap analyses

②

Literature review

Automated synthesis of hundreds of papers, extraction of methods and results

③

Data analysis & coding

AI writes analysis pipelines, fits models, generates figures

④

Writing & revision

AI drafts papers, rebuttals, grants. Often indistinguishable from human writing.

Observed effect when AI used uncritically: lower quality · less original · less time invested per result

Four values that define research integrity

Downstream

Societal benefit

Does it advance knowledge that helps people?

Epistemic

Robust knowledge

Is it reproducible, original, and rigorously validated?

Practice

Curiosity & judgment

Does it cultivate scientific thinking or outsource it?

Civic

Better scientists

Does it make better researchers and public communicators?

The answer is not "prohibit" or "allow freely." It is: allow considered use · hold yourself accountable · verify AI outputs with domain knowledge · revise norms as evidence arrives.
de Fine Licht · Chalmers · "Responsible Research in the Age of GenAI" · 2026

Honest uncertainty

Questions we don't yet have answers to

Being honest about uncertainty is part of the message. This room is on the actual frontier — not just of using AI, but of figuring out what it means for science.

Training

How do we build deep expertise?

If AI handles entry-level tasks, how do junior researchers develop the physical intuition those tasks were supposed to create? We don't have a good answer yet.

Quality

Where is the quality floor?

AI accelerates output — but if everyone uses it, average quality could drop while volume surges. What does rigorous AI-assisted science look like at scale?

Ownership

Who owns AI-assisted work?

Authorship, IP, and attribution norms for AI-assisted experiments, papers, and datasets are still being worked out across journals and universities — including KTH.

Equity

How unequal will the advantage be?

Groups with better tools and AI fluency will produce more. Does this widen the gap between well-resourced and under-resourced research communities worldwide?

These are not rhetorical questions — they are the reason this initiative exists. Today is the beginning of building answers together, not just acquiring fluency. The June 15 Teachers' Meeting and October Workshop will return to all of these.
