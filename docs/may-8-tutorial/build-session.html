Step 1

Brainstorm your idea with ChatGPT

Start by talking to ChatGPT. Copy the prompt below, paste it in, then answer its questions. Goal: land on something concrete and achievable in 30–40 minutes.

I am a researcher / faculty member at the Department of Applied Physics, KTH. I am attending a hands-on AI tutorial and have 45 minutes to build something useful with Claude Code — an AI agent that can write and run code, read my files, and execute multi-step tasks autonomously.


My research area: [fill in — e.g. "ultrafast laser spectroscopy" / "X-ray imaging" / "cell biophysics" / "nanophotonics"]


Help me brainstorm a mini-project that:
— Is achievable in 30–40 minutes with an AI agent
— Is genuinely useful to my research, teaching, or daily work
— Uses data or files I can provide (or synthetic examples if I have nothing handy)
— Produces a concrete output: a working script, a figure, a document, or a small tool


Ask me up to 4 clarifying questions to understand my situation. Then suggest 2–3 options ranked by feasibility. Once I pick one, write a concise 3-sentence project description I can paste into a CLAUDE.md file.

How to use this: Paste into ChatGPT, fill in your research area, and answer its follow-up questions. When you have a project description, copy it — you'll use it in Step 3.

Ideas to get you started

No data handy? Pick any of these — Claude Code can generate synthetic examples to test with.

LAMP · BON — Spectroscopy

Peak fitting & batch processing

Paste or upload a CSV of spectral data. Claude Code fits Gaussian peaks, flags anomalies, and writes a batch script for your full dataset. Output: cleaned data + publication-quality figures.

BON · Biophysics — Microscopy

Image analysis script

Provide a few microscopy images (or sample TIFs). Claude Code writes a Python script to count cells, measure areas, or segment structures. Output: measurements CSV + annotated images.

All divisions — Writing

Research summary generator

Drop in 3–5 abstracts or papers. Claude Code extracts key findings and drafts a structured 1-page narrative for a grant section or paper introduction.

All divisions — Data

Data explorer & visualiser

Give Claude Code any CSV or Excel file from your research. It profiles the data, produces exploratory plots, and suggests what analyses to run next.

LAMP · BON — Automation

Instrument control script

Describe your instrument's API or existing control code. Claude Code writes a Python wrapper with a measurement loop and documentation. Output: a working, reusable module.

Professors — Teaching

AI-resistant assignment design

Tell Claude Code your course and topic. It generates 3 assignment options requiring physical lab work, oral defence, or in-situ reasoning — things AI cannot shortcut for a student.

All divisions — Literature

Paper comparison table

Paste 3–5 abstracts or excerpts. Claude Code builds a structured table — methods, datasets, key results, limitations — ready to drop into your manuscript or grant.

All divisions — Productivity

Repetitive task automator

Describe something you do repeatedly: rename files, reformat data, generate weekly summaries. Claude Code writes a script that does it in one command.

Step 3

Write your CLAUDE.md

CLAUDE.md is read by Claude Code at the start of every conversation — it is your standing briefing for the agent. Keep it short and specific: only include things the agent cannot figure out by reading your code and data. See the official best practices for what to include and what to leave out.

The CLAUDE.md rule: For each line you write, ask "Would removing this cause Claude to make a mistake?" If not, cut it. Bloated files cause the agent to miss your real instructions.

Minimal template — paste this into a new file called CLAUDE.md

CLAUDE.md

# [Project name — 3–5 words]

## Goal
[Paste the 2–3 sentence description from your ChatGPT session here.]

## Context
- I am a [role] at Applied Physics, KTH working on [field]
- Key constraint: [one physical/domain constraint Claude must respect, e.g.
  "Background subtraction must use Shirley method — never set negative"]

## Data
- [filename]: [what it contains, format, units, any quirks]
- Or: no data yet — generate a realistic synthetic example first

## Output
- [What the finished result should be — script / figure / document]
- Prefer working over perfect; comment any non-obvious code

✅ Good things to put in CLAUDE.md

  • Your specific data format (column names, units)
  • A domain constraint Claude would get wrong (e.g. "Shirley background only")
  • The exact output you want (script / figure / doc)
  • How to run a verification check ("plot the residuals")

❌ Leave these out

  • Standard Python conventions — Claude already knows them
  • Long background explanations of your field
  • Instructions that repeat what the file names already say
  • Anything you'd write in a paper introduction
Shortcut — let ChatGPT write it: After your brainstorm, say: "Now write a concise CLAUDE.md for this project. Include only what the agent cannot figure out from the files themselves. Use this format: Goal / Context / Data / Output. Max 15 lines."
Step 4

Start building with Claude Code

Open your terminal in the project folder, start Claude Code, then paste the starting prompt below as your first message.

cd ~/my-project
claude

Paste this as your first message:

Read CLAUDE.md carefully — that is your full briefing.

First, ask me any clarifying questions you need about the data format, expected output, or constraints.

Then enter plan mode: propose a step-by-step plan (max 5 steps) and wait for my approval before writing any code.

We have ~30 minutes. Prioritize a working result over a perfect one.
What happens next: Claude Code reads your CLAUDE.md, asks clarifying questions, proposes a plan. Review it, approve (or adjust), then let it work. It will write code, run it, fix errors, and iterate — without you approving every step. Your job is to review the output and apply your domain judgment at the end.

Full workflow guide: code.claude.com/docs/en/best-practices

Techniques from today — applied

Use what you learned in the earlier sessions to get the most from this session.

Manager mindset

Step back and let it work

After approving the plan, stop watching every line. Let Claude loop — it thinks, writes, runs, checks output, and fixes errors autonomously. Interrupt only if it explicitly asks a question or clearly goes off track.

Domain knowledge

Correct with physics, not frustration

When output is wrong, explain why in physical terms: "In ARPES, the background is never negative — this fit is unphysical. Use a Shirley background." Your expertise is the correction signal the agent cannot generate itself.

Use both tools

Stuck? Switch to ChatGPT

If Claude Code hits an error you don't understand, paste the error into ChatGPT: "What does this Python error mean and how do I fix it?" Bring the answer back to Claude Code. The two tools work as a team.

Iterate, don't restart

Refine what exists

When a first version is done, layer improvements: "Add axis labels with units", "Accept a filename as a command-line argument", "Write a test that verifies the peak positions." Small iterations compound fast.

Useful Claude Code commands during the session

Esc — stop Claude mid-action and redirect
Esc Esc — rewind to a previous checkpoint
/clear — reset context if session gets noisy
/plan — enter plan mode (read + think, no edits)

Step 5

Share what you built

At 16:45 we come back together for a short show & tell. You don't need to have finished. A partial result with honest insight about the process is more valuable than a polished one with nothing to say.

Option A — Live demo

Screen-share (or project) what you built. Walk through what the agent did, what surprised you, and what you'd tackle next with more time. 2–3 minutes per group.

Option B — Zoom chat

Post a screenshot, GitHub link, or your claude.ai conversation link in the Zoom chat. Remote participants: this is your main option. To get a link: open the conversation in claude.ai → Share → Copy link.

Optional — submit to the APHYS AI gallery: If you'd like your use case to appear on the department AI website, paste this prompt into Claude Code at the end of your session:
Submit this project as a use case to the APHYS AI gallery at https://hypha.aicell.io/kth-sci/artifacts/aphys-ai-gallery. Include: title, 2-sentence description, my division [BON / LAMP / Biophysics / other], tagged "may8-tutorial".
