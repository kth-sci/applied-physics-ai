Tips from Colleagues

Quick insights and recommendations from faculty who are already using AI.

WO
Wei Ouyang • Biophysics / AI for Cell Biology

"Start by giving AI a task you already know how to do manually. That way, you can evaluate the output quality and learn how the tools work without risk. Once you trust your ability to judge the results, start using it for new tasks."

JS
Jonas Sellberg • X-ray Physics / Ultrafast Science

"When using AI for data analysis, always check the physics. The model might give you a mathematically correct fit that is physically meaningless. Your domain knowledge is the quality control. A PCA that explains 99% of variance is useless if the principal components have no physical interpretation."

FK
Faculty Contributor • Engineering Mechanics (organized original retreat)

"The biggest surprise from our retreat was how much faculty productivity improved in the weeks after -- not from fancy ML models, but from simple things like using ChatGPT for grant writing, email drafts, and code debugging. Start simple."

AK
Anna K. • 2 days ago • Research

Used Claude Code to process 200+ Raman spectra that had been sitting on my drive for months. Baseline correction, peak fitting, and a summary report -- all done in about 30 minutes. Would have taken me a full day manually. The key was being very specific about the fitting model (Voigt, not Gaussian) and the baseline method.

ML
Marcus L. • 5 days ago • Teaching

I tried having ChatGPT generate exam questions for my optics course. The first attempt was mediocre -- too textbook-like. When I added "create questions that test physical intuition, not just formula application" and gave it two example questions I liked, the quality improved dramatically. Context matters.

ES
Erik S. • 1 week ago • Productivity

Converted a 15-year-old Fortran simulation code to Python using Claude Code. It took about 2 hours of back-and-forth, but the results match to machine precision. The agent even added proper error handling and documentation that the original never had. Highly recommend for legacy code migration.
