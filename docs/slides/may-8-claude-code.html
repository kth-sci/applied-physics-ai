Recap

You already know the key idea.

ChatGPT

Text in. Text out.

  • Replies once, then waits
  • No access to your files
  • No memory between sessions
  • You copy-paste everything manually
  • Output lives only in the chat window
Claude Code

Acts in your project.

  • Reads your files — code, data, PDFs, notebooks
  • Runs terminal commands — python, bash, git, conda
  • Saves output directly to disk — figures, reports, scripts
  • Loops until the task is done — plan → act → fix → repeat
  • Persists instructions across sessions via CLAUDE.md
Today: you don't learn a new concept. You learn the tool. You already understand what an AI agent is. The next 45 minutes are about the specific commands, workflows, and use cases that make Claude Code productive for Applied Physics research.

Standing Instructions

One file. Every session.

CLAUDE.md is a plain-text file at the root of your project. Claude reads it at the start of every session — automatically, without you saying anything.

① Project context

What this folder is for, what data format you're working with, what tools and libraries are installed.

② Conventions

"Always save figures to ./figures/" · "Use seaborn-whitegrid style" · "Never modify raw data files"

③ Constraints

What NOT to do, which directories are off-limits, who to ask before deleting files, safety rails for the project.

# Pump-Probe Spectroscopy Analysis KTH LAMP group — ultrafast transient absorption data. Data: CSV files in ./data/ columns: time_ps, wavelength_nm, delta_OD Always use matplotlib with seaborn-whitegrid style. Save all figures to ./figures/ as 300 dpi PNG. Never modify files in ./raw_data/. Python environment: conda env 'spectro' (python 3.11, numpy, scipy, matplotlib).

/init generates a CLAUDE.md from your project automatically — run it on any existing folder to get a starting point.

The Key Mindset Shift

The question is not: what prompt should I write?
The question is: what should Claude know?

Context engineering — deliberately shaping what's in the context window — is the core skill of working with AI agents.

Context Window

What Claude sees right now

Everything Claude knows at this moment: your files, CLAUDE.md, conversation history, tool outputs. It does not know what you haven't shown it. A blank context produces generic output — even from the most capable model.

Your 10 years of domain expertise is invisible unless you put it in the context.
Three Levers

How to shape the context

CLAUDE.md — persistent across all sessions, set once; files you reference — open or mention specific files in your task; how you phrase the task — specificity matters even when context is rich.

Each lever compounds. All three together = expert-level output.
Rich Context → Rich Output

The compounding effect

A vague prompt with no file context produces textbook-generic text. A precise task with well-structured project files and a good CLAUDE.md produces working, domain-specific code and analysis.

"Analyse my data" → generic script. "Read pump_probe.csv, fit 550 nm, plot fit vs data" → working figure.
CLAUDE.md is the most powerful single action you can take before starting a task. Write it once; benefit every session. It is the equivalent of briefing a skilled postdoc before handing them a project.

What It Can Do

Tools Claude Code can use — and how to control them.

Read & Write Files

Your project, fully accessible

Reads any file in your directory — Python, CSV, TIFF, PDF, notebook. Writes, creates, and edits files with your confirmation before any destructive action.

Read ta_data.csv → write analysis.py → save figure_2a.png
Run Terminal Commands

Your shell, on demand

Executes python, bash, git, curl, conda — anything your shell can run. It reads the output, handles errors, and retries. You don't need to copy-paste error messages.

python fit_decay.py → read traceback → fix the code → re-run
Search the Web

Docs, papers, Stack Overflow

Built-in web search for documentation, recent papers, package references, and error solutions — when enabled. Searches in context of your actual task.

"Find the scipy.signal docs for savgol_filter and show me the right window_length argument"
MCP Tools

Connect to external apps

The Model Context Protocol connects Claude Code to databases, browsers, CAD tools, and lab instruments. Blender, Playwright, SQL — extensible by design. (More on this later.)

Browser automation · Blender scene control · database queries
Permission Modes
  • Default — asks before every action that changes files or runs commands
  • --dangerously-skip-permissions — auto-approve all actions; use only in trusted, version-controlled projects
Safety Controls
  • Esc — interrupt the running action immediately
  • Esc + Esc — undo the last file change
  • Always git commit before asking Claude to do anything risky

The Commands You Need

Six commands cover 90% of usage.

claude
Start an interactive session in the current directory — Claude reads the directory and CLAUDE.md immediately
claude "do this task"
One-shot: run a single task non-interactively and exit — useful for scripting and automation
/init
Generate a CLAUDE.md from your current project — Claude reads all files and writes standing instructions; run once per project
#add a convention
Inside a session: append a line to CLAUDE.md without executing a task — builds your instruction file incrementally as you work
Esc
Interrupt the current action mid-run — stops Claude without undoing any changes already made
Esc + Esc
Undo the last file change — restores the previous state of any file Claude just wrote or modified

Full reference: claude --help · or press / inside a session to see all slash commands

Use Case 04

SSH into your compute cluster. Fix the bug in context.

Remote Server

Your HPC cluster, conversational

Claude Code can SSH into a remote machine, navigate the filesystem, submit jobs to a scheduler (SLURM), monitor output files, and download results — all through natural language. Your compute cluster becomes as easy to use as a chat window.

  • "Submit the job array in ./scripts/sweep.sh with 16 tasks"
  • "Check the status of my running jobs and show me which ones finished"
  • "Download the output from run_42/ and plot the loss curve"
Bug Fixing

Full context, not just the error line

Paste the error message. Claude reads the full file — not just the snippet you showed. It understands the surrounding code, applies the fix, and asks you to confirm before writing. Not "here's how to fix it" — it actually fixes it.

  • Reads the full source file for context
  • Understands the data shape from earlier in the script
  • Applies the fix and confirms before writing
  • If the fix doesn't work, reads the new error and tries again

Advanced Extension

Ralph Loop: autonomous, iterative, self-scheduling.

A Ralph Loop is a structured agentic workflow where Claude Code self-paces iterations of a task — waking up periodically, checking state, acting, and sleeping again — until a condition is met.

The Pattern

Define · Act · Check · Repeat

Define a task with an explicit exit condition. Claude acts, checks state, schedules the next wake-up, and repeats. You only intervene when it surfaces a decision that requires your judgment.

While new_data_files: analyze → append to report → sleep 5 min → wake → check for new files → repeat
Physics Use Cases

Overnight experiments

Monitor an experiment data folder — when a new CSV appears, analyze it and append to the report. Run a parameter sweep overnight; surface only runs that meet a quality threshold in the morning.

Watch ./data/ → new pump_probe_*.csv → fit → append to summary.md → Slack notification
Caution

Autonomous = less oversight

Always set a clear exit condition — a loop without one runs forever. Review what Claude did before the next iteration. Never use for tasks with irreversible or high-stakes actions without review gates.

git commit before each iteration · define max_iterations · log every action to a file
Ralph Loop is how Wei runs this tutorial website. The daemon.py process monitors Slack and the Hypha API, relays messages to the active session, and acts on new registrations and feedback — autonomously. It ran during this tutorial.

Literature synthesis

From PDFs to structured summary

Setup: create a folder, drag in 3–5 PDFs from your reading pile. Takes 2 minutes to set up, 8 minutes to run.

Flag any contradictions between papers. Add a column for "open questions".
Data analysis

Your data, working code

"Read [your data file]. Describe its structure. Write a Python script that [your analysis]. Run it. If it errors, fix it. Save the figure as output.png."

Setup: navigate to a folder with a CSV, TIF stack, or PDB file. Claude will describe the data before writing any code.

Works for pump-probe CSV, fluorescence image stacks, crystallography data.
Writing task

Rewrite any paragraph

"Read my_draft.txt. Identify the weakest argument. Rewrite that paragraph for a Nature Methods audience. Suggest 3 supporting citations I should look up."

Setup: paste 2–3 paragraphs of your draft into a plain text file. Name it my_draft.txt.

Works for grant writing, methods sections, discussion paragraphs, cover letters.
