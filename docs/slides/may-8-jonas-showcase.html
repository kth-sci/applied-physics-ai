The AI Workflow

Every page built in conversation — no manual coding

The design brief, the HTML, the CSS, the JavaScript, the git commits, the GitHub push — all directed through natural language. Claude Code handles the rest.

Week 1
"Create a department AI webpage with Tailwind CSS, a hero section, and an events page." → First working site in one afternoon. No Tailwind knowledge required.
Week 2
"Add a use-case gallery that loads from the Hypha API. Cards with tags, difficulty, time saved." → Gallery wired to live data. Submissions appear automatically without touching the code.
Week 3
"Build a registration form for May 8. Cap at 60 onsite. Send confirmation emails when a seat is approved." → Full registration pipeline: form → Hypha → daemon → email. Zero manual handling.
This week
"Make the hero more subtle — match the color of the index page." / "Add a Zoom link button." / "Add one-sentence descriptions to the agenda." → Changes deployed in under 60 seconds each.
The key shift: you describe outcomes, not implementation. "The hero banner is too orange and alarming" is a valid instruction. You don't need to know CSS gradient syntax.

Beyond the Web

Claude Code
controls Blender.

24 hours ago, I had never used Blender. Blender is a free, open-source 3D creation suite covering modelling, animation, rendering, and simulation — used everywhere from scientific visualisation to film VFX.

What MCP is

An open protocol (Anthropic, 2024) for connecting AI agents to external tools — databases, browsers, CAD software, instruments.

Blender MCP server

An addon for Blender that exposes its Python API as an MCP server. Claude Code connects to it and issues commands directly.

What becomes possible

Scientific visualisations, molecular animations, instrument schematics, publication figures — described in words, built in Blender.

"Center it, make it into blue PLA plastic, render a looping gif where the object rotates around the z-axis in low resolution"

Try It Yourself

Set up Blender MCP in Claude Code — paste this prompt

Open Claude Code in a terminal and paste the prompt below. It will walk you through the full setup step by step, adapted to your OS.

I want to connect Claude Code to Blender using the Blender MCP server so I can control Blender with natural language. Please guide me through the full setup step by step, adapted to my system. First ask me: Q1. What OS am I on? (macOS / Windows / Linux) Q2. Do I have Blender installed? If yes, which version? Q3. Do I have Node.js installed? (run: node --version to check) Q4. Do I have Python 3.8+ available? (run: python3 --version) Then walk me through these steps in order, waiting for my confirmation after each before continuing: STEP 1 — Install Blender (if needed) · macOS: brew install --cask blender · Windows: winget install BlenderFoundation.Blender · Linux: sudo snap install blender --classic · Or download from https://www.blender.org/download/ Target version: 4.x (recommended) STEP 2 — Install the Blender MCP addon · Download the addon zip from: https://github.com/ahujasid/blender-mcp/releases/latest · In Blender: Edit → Preferences → Add-ons → Install · Select the downloaded .zip file · Enable the addon (check the box) · Verify: a "Blender MCP" panel should appear in the N-panel of the 3D Viewport (press N to open) STEP 3 — Start the MCP server inside Blender · In the 3D Viewport, press N to open the side panel · Click the "Blender MCP" tab · Click "Start MCP Server" · The status should change to "Server running on port 9876" · Keep Blender open — it must stay running for Claude Code to connect STEP 4 — Configure Claude Code to connect Option A — using the CLI (recommended): claude mcp add blender --transport sse \ --url http://localhost:9876/sse Option B — edit settings.json directly: Open (or create) .claude/settings.json and add: { "mcpServers": { "blender": { "transport": "sse", "url": "http://localhost:9876/sse" } } } STEP 5 — Test the connection · In Claude Code, type: /mcp · You should see "blender" listed as a connected server · Then type: "Create a simple scene in Blender: a red sphere on a grey plane, with a sun lamp. Position the camera at (5, -5, 3) looking at the origin." · Watch Blender update in real time If any step fails, paste the exact error message and I will diagnose it. Begin with Q1.

First-timer Perspective

Three things I did not expect

① Speed

The pace is disorienting

What I expected to take a week took an afternoon. The bottleneck is no longer writing code — it is deciding what to build and whether the result is correct.

This is Karpathy's point: "You can outsource your thinking, but you cannot outsource your understanding."

② Iteration

Vague instructions work — to a point

"Make it look nicer" produces something. "Make the hero match the slate-900 nav background, remove the orange gradient, keep the white text" produces exactly what I meant.

Domain precision matters. The prompting skills from Jonas's session directly apply here.

③ Judgment

Your expertise is not optional

The agent made confident mistakes — wrong assumptions about the audience, incorrect API calls, outdated library syntax. Catching those requires knowing your field.

Applied physicists have strong judgment. That is the skill that scales with AI — not the typing.
