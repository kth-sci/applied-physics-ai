AI Agents: Your Autonomous Research Assistant

Move beyond chat -- let AI plan, execute, and iterate on complex tasks for you.

What Is an AI Agent?

The leap from asking questions to delegating tasks.

Chatbot (ChatGPT)

You ask
AI responds
You ask again
AI responds

Reactive: one question at a time

AI Agent (Claude Code)

Plan
Execute
Evaluate

Iterate until done

Files Code Terminal Web Memory

A chatbot like ChatGPT responds to your messages one at a time. You prompt, it replies, you prompt again. It is reactive.

An AI agent like Claude Code operates differently. You describe a goal -- "analyze my spectroscopy data and create a report" -- and the agent autonomously plans the steps, reads your files, writes code, runs it, evaluates the results, and iterates until the task is complete. It has access to your filesystem, your terminal, and the web.

The analogy: ChatGPT is like emailing a colleague a question. Claude Code is like having a research assistant sit at your computer and do the work -- while you supervise the approach and evaluate the output.

Key Concepts

Tool Use

AI agents can use tools: read and write files, execute shell commands, search the web, run Python scripts. This is what makes them powerful -- they can interact with your actual work environment.

Planning & Iteration

Given a complex task, the agent breaks it into steps, executes each one, evaluates the result, and adjusts. If code fails, it reads the error, fixes it, and tries again -- just like you would debug.

Memory & Context

Claude Code maintains memory across sessions. It remembers your project structure, your preferences, and the context of your work. You can teach it once and it retains that knowledge for future sessions.

Human Oversight

You remain in control. The agent asks for permission before executing potentially impactful commands. You review its work, approve changes, and guide its direction. Think of it as supervised autonomy.

Getting Started with Claude Code

From zero to your first AI agent task in 15 minutes.

Step 1: Get a Claude Account

  1. Go to claude.ai and create an account (or log in if you have one)
  2. Subscribe to Claude Pro ($20/month) -- this gives you access to Claude Code
  3. For heavy use (large codebases, many hours/day), consider Claude Max ($100/month)

Step 2: Install Claude Code

Claude Code runs in your terminal. You need Node.js installed (version 18+).

# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Navigate to your project directory
cd ~/my-research-project

# Start Claude Code
claude

On first run, it will open a browser window for authentication. Log in with your Anthropic account.

Step 3: Give It a Task

Once inside Claude Code, simply describe what you want in natural language:

# Example: analyze data
> Read the CSV file in data/spectra.csv and create a plot of all spectra.
  Identify the main peaks and label them.

# Example: build a tool
> Create a Python script that batch-processes all .tif microscopy images
  in the images/ folder, applies background subtraction, and saves the
  results as PNG files with a scale bar.

# Example: convert code
> Convert the MATLAB script analysis.m to Python. Use numpy and scipy.
  Make sure the numerical results match.
That is it. Claude Code will read the files, plan the approach, write the code, run it, check the output, and iterate. You review and approve each step.

What Can You Build? Real Examples

Actual tasks that faculty in Applied Physics can accomplish with Claude Code.

Analyze Spectroscopy Data

"Read all .csv files in data/raman/. Each file has columns wavelength and intensity. For each spectrum: subtract the baseline using asymmetric least squares, find all peaks above 3x the noise floor, fit each peak with a Voigt profile, and create a summary table with peak positions, widths, and areas. Memory: Teaching Your Agent

Claude Code remembers your preferences and project context across sessions.

Unlike a chatbot where each conversation starts fresh, Claude Code maintains persistent memory. It stores project-level instructions in a CLAUDE.md file in your repository, which it reads at the start of every session.

What It Remembers

  • Your project structure and conventions
  • Coding style preferences (Python version, libraries)
  • Common file locations and data formats
  • Previous decisions and their rationale
  • Your feedback ("always use scipy, not manual FFT")

How to Teach It

  • Say "Remember that..." to store preferences
  • Edit CLAUDE.md directly with project-wide instructions
  • Correct mistakes -- it learns from your feedback
  • The more you use it, the better it gets at your specific workflows
# Example CLAUDE.md for a research project
# My Research Project

## Context
This is an ultrafast spectroscopy data analysis project.
Data files are in data/ as HDF5 format.
Use Python 3.11+ with numpy, scipy, matplotlib, and h5py.

## Conventions
- All plots should use the "science" matplotlib style
- Wavelength in nm, time delays in fs
- Save figures as both PNG (300 dpi) and SVG
